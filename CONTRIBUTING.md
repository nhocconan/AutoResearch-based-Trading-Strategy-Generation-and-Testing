# Contributing

Thanks for your interest! This is an autonomous trading-strategy research system
(an [autoresearch](https://github.com/karpathy/autoresearch)-style LLM loop). The
project lives or dies on one principle:

> **The thing being evaluated cannot make the evaluation easier on itself.**

The LLM agent writes strategies, but the simulation that judges them —
`backtest.py`, `evaluate.py`, `research_rules.py` — is **immutable**. The same
discipline applies to human contributors. Read this before opening a PR.

## Quick start

```bash
git clone https://github.com/nhocconan/trading-llm-auto-research.git
cd trading-llm-auto-research
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements-ci.txt   # minimal: enough to run the tests
pytest                                # 47 tests, < 1s, no data download needed
```

For the full system (live research loop, dashboard, real data) install
`requirements.txt` and follow the [README Quick Start](README.md#quick-start).

## The golden rules

1. **No look-ahead bias, ever.** At bar `i` a strategy may only use
   `prices.iloc[:i+1]`. No negative `.shift(-n)`, no `prices[i+1]`, no
   `.resample()` to fake a higher timeframe. The [validator](validator.py)
   rejects these statically, and the engine shifts every signal by one bar so a
   signal at `t` can only fill at `t+1` open.
2. **Use real higher-timeframe data.** Load HTF via
   [`mtf_data.get_htf_data()`](mtf_data.py) — never resample. HTF values are
   only visible to a lower-timeframe bar *after that candle closes*.
3. **Honest costs.** 0.04% taker + 0.01% slippage = 0.10% round trip, plus
   funding every 8h. These are enforced by the engine and pinned by a test.
4. **The engine is immutable.** If you genuinely need to change
   `backtest.py` / `evaluate.py` / `research_rules.py`, you must (a) explain why
   in the PR, (b) add or update tests that prove the new behaviour, and (c) note
   the change in [CHANGELOG.md](CHANGELOG.md) — existing results were computed
   under the old behaviour.

## Adding a strategy idea

Strategies are generated by the LLM from the knowledge base in
[`program.md`](program.md). The best way to contribute trading ideas is to add a
well-described technique there (entry/exit logic, indicators, position sizing,
the market regime it targets). A strategy is *kept* only if it passes the
per-symbol train **and** out-of-sample test gate (see the README diagram).

A hand-written `strategy.py` must define:

```python
name = "my_strategy_v1"
timeframe = "4h"          # one of 5m,15m,30m,1h,4h,6h,12h,1d,1w (1m banned)
leverage = 1.0            # ≤ 5.0; default 1.0

def generate_signals(prices):       # prices: OHLCV DataFrame
    # return a numpy array, same length as prices.
    # values are TARGET position sizes in [-0.40, 0.40]; 0 = flat.
    ...
```

Validate it before anything else:

```bash
python validator.py strategy.py
```

## Tests & CI

CI runs lint + validator + `pytest` + a results-integrity check on Python 3.11
and 3.12. Before pushing:

```bash
ruff check tests/ scripts/
pytest
```

If you change engine behaviour, **add the test first**. See
[`tests/README.md`](tests/README.md) for what each test file guards. All tests
use synthetic data, so they run anywhere with no market-data download.

## Pull requests

- Keep PRs focused; one concern per PR.
- Describe *what* changed and *why*, and call out anything touching the engine.
- Make sure CI is green.
